# Hermes Agent 文档学习方法论

> 如何用最少的时间吃透这 9 篇文档，达到面试准备标准

---

## 核心原则

**不要只读文档，也不要逐行翻代码。用"三层递进法"。**

- 文档 77,000 行源码的浓缩精华 —— 但光读不练记不住
- 源码是验证工具 —— 不是阅读对象，只读文档指引的关键路径
- 面试题是最佳学习检验 —— 遮住答案先自己作答

---

## 一、三层递进法

### 第一层：鸟瞰（每篇 30 分钟）

**目标**：建立心智模型，不求懂细节

1. **先看 00-项目总览**（15 分钟）
   - 看整体架构图，理解模块分层
   - 看核心数据流，理解消息的完整路径
   - 看运行时模式对比，理解系统运行方式

2. **按顺序速读 01-08**，每篇只看：
   - 模块概述（第一部分）—— 这个模块干什么
   - 架构图/ASCII 图 —— 内部结构
   - 面试题列表（不读答案）—— 应该能回答什么

3. **自测**：合上文档，用白纸画出你能回忆出的组件关系图
   - 能画出 Agent → 工具 → LLM 的三角关系吗？
   - 能写出 Gateway 的 5 层处理流程吗？
   - 能区分 MCP 和 ACP 的定位吗？

> 这一步结束，你应该能说出：项目分几层、每个模块干什么、数据怎么流动

---

### 第二层：深潜（每篇 2-3 小时）

**目标**：理解关键实现细节，能回答面试题

**每篇文档的标准流程**：

```
┌─────────────────────────────────────────────────────┐
│  Step 1: 精读文档全文                    45 分钟    │
│  Step 2: 打开源码读关键函数              30 分钟    │
│  Step 3: 遮答案自答面试题                45 分钟    │
│  Step 4: 画一张时序图/流程图              30 分钟    │
└─────────────────────────────────────────────────────┘
```

**Step 2 的关键原则**：代码只看文档引用的函数，不要通读源文件。

| 文档 | 只看这些函数/文件（用 grep 跳转） |
|------|----------------------------------|
| 01 | `run_agent.py`: `run_conversation()`, `_execute_tool_calls()`, `_handle_max_iterations()` |
| 02 | `tools/registry.py`: `register()`, `dispatch()`; `model_tools.py`: `handle_function_call()` |
| 03 | `gateway/run.py`: `start_gateway()`; `gateway/platforms/base.py`: `handle_message()` |
| 04 | `cli.py`: `process_loop()`, `chat()`, `_cprint()` |
| 05 | `tools/memory_tool.py`: `MemoryTool`; `hermes_state.py`: `SessionDB` |
| 06 | `tools/mcp_tool.py`: `MCPServerTask`; `acp_adapter/server.py`: `HermesACPAgent` |
| 07 | `agent/prompt_builder.py`: `_build_system_prompt()`; `agent/context_compressor.py`: `compress()` |
| 08 | `hermes_cli/plugins.py`: `PluginCollector`; `hermes_state.py`: `SessionDB` |

**Step 3 的正确姿势**：

1. 挑 5 道面试题
2. 用纸遮住答案
3. 对着空气大声说一遍你的回答（至少 2 分钟）
4. 然后看答案，标记你说漏/说错的地方
5. 重新说一遍直到流畅

---

### 第三层：串联（每篇 1 小时）

**目标**：能跨模块回答问题，达到面试要求

**每篇文档问自己 3 个串联问题**，用白话复述一遍完整链路：

| 文档 | 必问自己的串联题 |
|------|----------------|
| 01 | 如果 LLM 调用失败，从 Agent 循环到基础设施，总共经过哪些恢复层？ |
| 02 | 从写一个新工具到 Agent 能调用它，经过哪些模块？画全路径 |
| 03 | 一条 Telegram 消息从入站到用户收到回复，经过哪些系统？ |
| 04 | CLI 收到 /background 命令后，Agent 怎么在不同线程间通信？ |
| 05 | 用户问"我昨天说的那个 bug 还记得吗"，记忆系统怎么检索？ |
| 06 | 外部 MCP 工具和内置工具的调用路径有什么不同？ |
| 07 | 一次 API 调用中，Prompt Caching 怎么生效？分几层缓存？ |
| 08 | 从 git push 到 Docker 镜像上线，CI 流水线怎么跑？ |

## 三、避坑指南

### 不要做的事

1. **不要逐行读 run_agent.py** —— 11,500 行你会疯的。文档已经帮你筛选了关键路径
2. **不要在没看完 00 的情况下开始 01** —— 你会迷失在细节里，不知道每个模块在整体中的位置
3. **不要只看文档不碰代码** —— 至少用 grep 跳到关键函数看 5 分钟，知道代码长什么样
4. **不要在凌晨学 Agent 循环** —— 状态机/异步桥接这些概念需要清醒的大脑

### 要做的事

1. **面试题遮住答案先自答** —— 这是最高效的学习检验方式，没有之一
2. **遇到不理解的概念立刻 grep 源码验证** —— 但只看 grep 出来的那几行，不要发散
3. **用自己的话复述** —— 如果你不能用大白话解释一个概念，说明你没真懂
4. **画图** —— ASCII 图也好，手绘也好，画出来你就记住了
5. **每天最多 2 个模块的精读** —— 超过会消化不良，边际效益递减

## 四、实用命令速查

```bash
# 快速跳转到文档引用的关键函数
grep -n "def run_conversation\|def _execute_tool_calls\|def _handle_max_iterations" run_agent.py

# 查看某个类的继承关系（适配器、Provider 等）
grep -n "class.*Adapter\|class.*Provider" gateway/platforms/base.py

# 理解某个配置项的影响范围
grep -rn "CONTEXT_COMPRESSION_THRESHOLD" .

# 看某个工具的完整实现
grep -n "def handler\|def register\|registry.register" tools/memory_tool.py

# 查找错误分类逻辑
grep -n "class.*Error\|FailoverReason\|classify" agent/error_classifier.py

# 快速统计模块规模（了解复杂度）
wc -l run_agent.py cli.py gateway/run.py
find tools -name "*.py" | wc -l
```

## 六、常见问题

**Q: 我需要先学 Python 异步编程吗？**
不需要深入。理解 `async/await` 和 `queue.Queue` 的基本概念即可，文档会解释关键桥接模式。

**Q: 我需要了解所有 17 个消息平台的 API 吗？**
完全不需要。了解 Telegram 和 Discord 两个适配器就足以理解 adapter 模式的通用设计。

**Q: 源码有 77,000 行，我该读哪些？**
按照文档指引，只读每个模块的关键函数（上面表格列出的那些）。不超过 2,000 行。

**Q: 面试官会不会问文档没写到的东西？**
有可能。这就是为什么要做第三层"串联"练习——它训练你推导未知问题的能力。

**Q: 文档里代码是简化版还是真实代码？**
经过本轮修复后，文档中的代码片段均已对照真实源码校正。但为了可读性，部分代码做了适度精简（去掉了不重要的边界处理）。
