**Hermes Agent** 是一个 **自进化的 AI Agent 框架**——支持 17+ 消息平台接入，具备记忆/技能学习系统，可通过 CLI/Gateway/ACP/MCP 多种方式运行，代码规模约 77,000 行。

## 二、项目功能全景：Hermes Agent 能做什么？

### 2.1 一句话总结

**Hermes Agent 是一个能使用工具、拥有记忆、会自我进化的 AI 助手**——你可以把它部署到任何地方（终端、聊天软件、编辑器、其他 Agent），它会像人类助手一样：记住你说过的话、使用各种工具完成任务、从经验中学习新技能。

### 2.3 具体使用场景

#### 场景一：个人 AI 助手（CLI 模式）

你在终端里和 Hermes 对话，它帮你写代码、查文档、搜网页、执行命令。

```bash
# 交互式对话
hermes

# 单次问答（适合脚本/管道）
hermes -q "帮我解释这个正则表达式：r'(?P<name>[w-]+)-(?P<version>d+.d+.d+)'"

# 指定模型
hermes -q "写一个快速排序" --model claude-sonnet-4-6
```

**能做的事**：
- 阅读/修改你本地的文件（有路径范围的 sandbox 限制）
- 执行终端命令（bash、python 等）
- 搜索网页、抓取网页内容
- 生成图片（通过 fal.ai 等）
- 语音输入/输出（本地 Whisper + Edge TTS）
- 分解复杂任务给子 Agent 并行执行

#### 场景二：聊天机器人（Gateway 模式）

部署一个 Hermes Agent 到消息平台，用户通过聊天软件与 AI 交互。

```bash
# 启动 Gateway，连接所有配置好的平台
hermes gateway start
```

**支持的平台**：Telegram、Discord、Slack、飞书、钉钉、Matrix、WhatsApp、iMessage、Signal、Microsoft Teams、Google Chat、LINE、KakaoTalk、Zalo、VK、IRC、X（Twitter）、BlueSky

**典型用法**：
- **团队知识库机器人**：部署到公司 Slack，回答团队问题，自动搜索内部文档
- **个人助理机器人**：部署到 Telegram，随时随地使用 AI，跨设备同步对话记忆
- **客服机器人**：处理常见问题，需要时升级给人工

#### 场景三：编辑器集成（ACP 模式）

通过 ACP（Agent Client Protocol）将 Hermes 集成到 VS Code 等编辑器。

```bash
hermes acp
```

**能力**：编辑器可以直接调用 Agent 进行代码审查、重构建议、自动补全、跨文件理解。

#### 场景四：Agent 间协作（MCP 模式）

将 Hermes 作为一个 MCP Server 暴露给其他 Agent 调用。

```bash
hermes mcp serve
```

**场景**：一个协调 Agent 将子任务分发给多个 Hermes 实例并行执行，每个实例有独立的工具集和记忆。

#### 场景五：批量评测（Batch 模式）

对数据集并发评测 Agent 表现。

```bash
python batch_runner.py --dataset swe-bench --concurrency 4
```

**用途**：评估不同模型/配置的 Agent 性能，用于研究和选型。

### 2.5 记忆与学习系统

这是 Hermes 区别于普通 LLM 应用的核心特性：

```
┌─────────────────────────────────────────────────────────────────┐
│                      记忆与学习流程                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   用户说："帮我查一下那个 bug 修复了吗"                          │
│                                                                 │
│   ┌──────────┐    ┌──────────┐    ┌──────────────┐             │
│   │ 1. 自动  │───▶│ 2. FTS5  │───▶│ 3. 相关记忆  │             │
│   │ 记忆预取 │    │ 全文搜索 │    │ 注入 Prompt  │             │
│   └──────────┘    └──────────┘    └──────────────┘             │
│                                                                 │
│   对话结束后：                                                   │
│                                                                 │
│   ┌──────────┐    ┌──────────┐    ┌──────────────┐             │
│   │ 4. 自动  │───▶│ 5. 提取  │───▶│ 6. 下次对话  │             │
│   │ 同步记忆 │    │ 可复用技能│    │ 自动加载技能 │             │
│   └──────────┘    └──────────┘    └──────────────┘             │
│                                                                 │
│   技能热更新：写完 skill 文件后 Agent 立即可用，无需重启          │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

**记忆类型**：
- **内置记忆**：SQLite + FTS5 全文搜索，自动存储对话摘要和关键信息
- **外部记忆 Provider**：支持 Honcho、Mem0 等专业记忆服务作为插件接入
- **技能系统**：Agent 从成功对话中自动提取可复用的操作流程，保存为 skill 文件

### 2.6 自进化能力

这是 "self-improving" 的真正含义：

```
用户教 Agent 完成一个任务
        │
        ▼
Agent 成功执行后，自动或手动将流程保存为 skill
        │
        ▼
skill 包含：元数据（何时用）+ 指令（怎么做）+ 参考文件
        │
        ▼
下次遇到类似请求，Agent 自动匹配并加载该 skill
        │
        ▼
skill 在使用中被优化（渐进式改进）
```

**实际例子**：
1. 你教 Agent "帮我部署这个项目到服务器"，它学会了部署流程
2. Agent 将部署流程保存为 `deploy-to-server` skill
3. 下次你说"部署"，Agent 自动加载该 skill，按你上次教它的方式执行
4. 如果部署脚本有变化，Agent 在使用中更新 skill 内容

## 五、运行时模式对比

| 模式 | 入口 | 说明 | 并发模型 | 适用场景 |
|------|------|------|---------|---------|
| **交互式 CLI** | `hermes` | prompt_toolkit TUI，支持流式输出、皮肤、斜杠命令 | 主线程(asycio) + 后台线程(执行) | 本地开发/调试 |
| **单次 CLI** | `hermes -q "..."` | 无交互，结果输出到 stdout | 单线程阻塞 | 脚本/管道 |
| **Gateway** | `hermes gateway start` | 长驻守进程，连接 17+ 消息平台 | 每消息创建线程 | 生产部署 |
| **ACP** | `hermes acp` | JSON-RPC over stdio | 每请求独立 session | 编辑器集成 |
| **MCP Server** | `hermes mcp serve` | 暴露对话为 MCP 工具 | FastMCP stdio | Agent-to-Agent |
| **Batch** | `batch_runner.py` | 数据集并发评测 | ThreadPool | 评估/训练 |
| **Docker** | `docker/Dockerfile` | 容器化部署 | 同各模式 | 云部署 |

## 七、关键技术决策一览

| 决策 | 选择 | 原因 |
|------|------|------|
| Agent 循环范式 | ReAct (Reasoning + Acting) | LLM 原生支持 tool_call，业界标准 |
| 工具注册 | AST 扫描 + 自注册 | 无需手动维护工具列表，添加工具只需写 handler |
| API 架构 | OpenAI 格式为规范，适配器翻译 | 一套内部格式，适配所有 LLM provider |
| 并发模型(CLI) | 主线程 asyncio + 后台 thread | prompt_toolkit 需要主线程控制终端 |
| 并发模型(Gateway) | 每消息创建 Agent 线程 | 隔离故障，简单可靠 |
| 消息去重 | TTL 缓存 + 持久化(飞书) | 多平台消息重放防护 |
| 记忆存储 | SQLite + FTS5 全文搜索 | 零依赖，嵌入式中使用 BM25 变体默认排序 |
| 流式输出 | sync queue.Queue 桥接线程 | 安全跨越 thread→asyncio 边界 |
| 上下文压缩 | LLM 摘要 + 多阶段 | 先裁剪工具输出，再 LLM 摘要中间消息 |
| 错误恢复 | 分类 → 退避 → 故障转移 → 凭证轮换 | 多层防御，自动恢复 |
| 插件系统 | entry-point + 目录扫描 | pip 安装 + 手动放入均可 |
| 技能系统 | 渐进式披露 (元数据→指令→文件) | 控制 token 消耗，按需加载 |
| 部署 | Docker 多架构 (amd64+arm64) | 支持 Apple Silicon / 云服务器 |
