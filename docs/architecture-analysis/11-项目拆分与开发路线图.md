# 11 - 项目拆分与 6-7 个月开发路线图

## 一、项目定位

### 项目名称建议: **DevAgent** (开发智能体)

### 核心定位
基于 hermes-agent 架构，拆出一个**多Agent软件开发自动化系统**，通过飞书/企微对话驱动，让多个AI Agent协作完成软件开发任务。

### 一句话描述
> 通过聊天工具下达开发任务，多个AI Agent自动协作完成：需求分析、代码编写、测试执行、部署上线。

## 六、关键技术难点与解决方案

### 6.1 多Agent状态同步
**难点**: 多个子Agent并行执行时，状态如何同步到飞书/企微?
**方案**: 
- 基于 hermes 的心跳机制 (30秒间隔)
- 飞书用卡片消息更新 (PATCH更新同一张卡片)
- 企微用模板卡片更新

### 6.2 长任务超时
**难点**: 飞书/企微消息有超时限制，复杂开发任务可能执行几十分钟
**方案**:
- 立即回复"收到，正在处理..."
- 定期更新进度卡片
- 完成后发送结果卡片

### 6.3 多模型API兼容
**难点**: 不同模型的API格式不同 (OpenAI/Anthropic/国产)
**方案**:
- 基于 hermes 的 API 模式检测
- 统一到 OpenAI 兼容格式 (DeepSeek/通义千问都支持)
- Anthropic 用适配器转换

### 6.4 代码执行安全
**难点**: Agent执行的代码可能有危险操作
**方案**:
- 基于 hermes 的 approval.py
- 飞书/企微卡片审批
- 命令白名单/黑名单
- Docker沙箱执行

### 6.5 上下文窗口管理
**难点**: 长对话 + 代码文件容易超出上下文限制
**方案**:
- 直接复用 hermes 的 ContextCompressor
- 工具输出截断
- 代码文件智能摘要
